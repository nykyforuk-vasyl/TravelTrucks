# TravelTrucks

# Project Description

üèÖ _TravelTrucks_ is the frontend part of a web application for a company specializing in camper rentals. The application allows users to browse a catalog of available campers, filter them by various parameters, view details of selected campers, leave reviews, and book transport.

The project is developed according to the technical specification, with additional improvements and changes.

# üöÄ Features

- üìå **Home Page** ‚Äì general information about the service.
- üöê **Camper Catalog** ‚Äì browse available vehicles.
- üîç **Filtering** ‚Äì search campers by location, features, and transport type in a modal window.
- üíñ **Favorites** ‚Äì ability to add campers to a favorites list (saved in localStorage).
- üî∫ **Scroll Up** ‚Äì quick and convenient return to the filter selection.
- üìÑ **Camper Details Page** ‚Äì detailed information about each vehicle, with a "Show More" button.
- ‚öô **Load More Cards** ‚Äì with the "Load More" button.
- ‚ú® **User Reviews** ‚Äì with a five-star rating system.
- üìÖ **Booking Form** ‚Äì the ability to submit a booking request.

# üõ†Ô∏è Technologies

- **React + Vite** ‚Äì fast development and efficient rendering of components.
- **Redux Toolkit** ‚Äì state management.
- **React Router** ‚Äì routing.
- **TailwindCSS** ‚Äì fast and efficient styling.
- **Axios ‚Äì API** requests.
- **Vercel** ‚Äì deployment.

# Main Pages

- **Home** (/) ‚Äì contains a banner and the main call-to-action.
- **Camper Catalog** (/catalog) ‚Äì a list of campers with filtering options and the ability to add to favorites.
- **Single Camper Page** (/catalog/:id) ‚Äì detailed information, gallery, reviews, and booking form.

# API

Data is fetched from the backend: MockAPI

- Main endpoints:

- **GET /campers** ‚Äì fetch list of campers

- **GET /campers/:id** ‚Äì fetch details of a specific camper
